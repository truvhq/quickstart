The admin process is now complete. Here's what happened. What data was returned?

Initializing the Citadel Bridge

  1. The quickstart front end requested a bridge_token from the quickstart back end.
  2. The quickstart back end requested a bridge_token from Citadel, providing admin as the value for product_type.
  3. Citadel provided the quickstart backend with a bridge_token, which the quickstart back end returned to the quickstart front end.
  4. The quickstart front end initialized the Citadel Bridge with the provided bridge_token

Connecting the administrator to their payroll administration account

  1. The administrator logged into their payroll administration account through the Citadel Bridge.
  2. The Citadel Bridge provided a public_token to the quickstart front end, who passed that public_token to the quickstart back end.
  3. The quickstart back end exchanged that public_token with Citadel for an access_token and stored that access_token for later use.

Accessing admin data from Citadel

  1. The quickstart back end uses the stored access_token to request a payroll report from Citadel.
  2. Citadel provides that admin information to the quickstart back end who sends it to the quickstart front end.
